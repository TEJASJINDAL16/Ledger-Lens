# Data Dictionary: LedgerLens Gold Warehouse

This document defines the schema of the final DuckDB Star Schema (`data/warehouse/ledgerlens.duckdb`), specifically the core Fact table and Merchant Dimension.

## Table: `fct_transactions`

| Column | Type | Description |
|---|---|---|
| `transaction_id` | VARCHAR | Primary key (UUID) for the transaction. |
| `account_id` | VARCHAR | Foreign key to `dim_account`. |
| `merchant_id` | VARCHAR | Foreign key to `dim_merchant`. Null if unresolved. |
| `date_key` | DATE | Foreign key to `dim_date`. |
| `txn_timestamp` | TIMESTAMP | The exact parsed UTC timestamp of the transaction. |
| `amount_inr` | DOUBLE | The transaction amount mathematically converted to INR. Guaranteed numeric. |
| `amount_original` | VARCHAR | The raw, unparsed amount string from the source system (e.g. `(45.00)`). Preserved for lineage. |
| `currency_original` | VARCHAR | The raw currency code. Preserved for lineage. |
| `raw_descriptor` | VARCHAR | The exact merchant string received from the payment gateway. |
| `descriptor_clean` | VARCHAR | The regex-normalized merchant string (stripped of state codes and store IDs) used for matching. |
| `channel` | VARCHAR | The transaction channel (e.g., `POS`, `ONLINE`, `ATM`). |
| `status` | VARCHAR | Transaction status (e.g., `APPROVED`, `DECLINED`). |
| `city_canonical` | VARCHAR | The cleanly mapped city name (e.g., `New Delhi` instead of `del`). |
| `country` | VARCHAR | The transaction country. |

### Data Health Flags (fct_transactions)
*These flags are crucial for honest analytics. Downstream consumers must check them.*

| Column | Type | Description |
|---|---|---|
| `match_method` | VARCHAR | How the merchant was resolved: `exact`, `fuzzy`, `cluster`, or `unresolved`. |
| `match_confidence` | DOUBLE | A score [0.0 - 1.0] indicating the statistical confidence of the resolution. |
| `mcc_imputed_flag` | BOOLEAN | `TRUE` if the MCC was originally missing and was algorithmically guessed. |
| `is_near_duplicate` | BOOLEAN | `TRUE` if this transaction occurred within 120s of an identical charge on the same account. **Do not aggregate blindly.** |
| `is_outlier` | BOOLEAN | `TRUE` if the transaction value is > 3.5 Median Absolute Deviations (MAD) from the merchant's median ticket size. |

## Table: `dim_merchant`

| Column | Type | Description |
|---|---|---|
| `merchant_id` | VARCHAR | Primary key (e.g., `M014`). Generated by the resolver. |
| `merchant_name` | VARCHAR | The canonical, human-readable brand name (e.g., `Uber Eats`). |
| `mcc` | VARCHAR | The 4-digit Merchant Category Code. |
| `merchant_category` | VARCHAR | The plain-text category mapped from the MCC (e.g., `Food Delivery`). |
| `category_group` | VARCHAR | The macro category group (e.g., `Dining`). |
